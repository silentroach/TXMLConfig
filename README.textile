A simple way to store your config in xml files. I have not a lot of time so this project is not updated recently, it will be great if you send me pull requests.

Features:

* published properties storage in a simple way without read & write methods coding
* objects storage (TXMLConfigNode)
* object list storage (TXMLConfigNodes)
* simple string encryption when EncryptedString? attribute is defined before property

----------

Простой способ для хранения конфигурации приложений в xml-файлах. Проектом занимаюсь довольно редко, поэтому любая помощь не помешает - присылайте пулл-реквесты.

Что можем:

* хранить published-свойства без необходимости реализовывать чтение значений и их запись
* хранить вложенные объекты (TXMLConfigNode)
* хранить списки объектов (TXMLConfigNodes)
* шифрование строковых свойств с атрибутом EncryptedString?
